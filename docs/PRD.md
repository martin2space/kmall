## 一、产品概述

### 一句话定位
给快猫女装门店管理者用的，数字化店员培训、排班、日常考核与绩效评比的内部管理系统。

### 背景与痛点
- 门店扩张中，店员培训靠口头传授，标准不统一
- 工作细则散落在纸质文件或微信群，店员查阅不方便
- 考核没有系统记录，打分靠印象，说不清楚合格不合格在哪
- 店员不知道自己哪里做得好、哪里需要改进，反馈滞后
- 目前门店依赖口头传达或纸质文档进行工作交代，存在信息传递损耗、工作执行不到位、考核不透明等痛点。

### 解决方案
一个轻量 Web 系统，双角色（管理员 + 店员），覆盖：
工作细则查阅 → 排班安排 → 日常任务执行 → 考核打分 → 绩效反馈 完整闭环。

### 核心目标

- **标准化：** 建立统一的工作细则库，让店员随时随地查阅。  
    
- **数字化：** 将排班与任务自动关联，实现“人、时、事”的精准匹配。  
    
- **闭环化：** 通过“执行-评分-反馈”的闭环，提升店员专业素养和工作积极性。

## 二、用户角色

### 角色 A：管理员（店长 / Martin）
- 维护工作细则库和考核标准
- 管理门店和店员信息
- 创建排班计划，分配每日任务
- 对店员进行考核打分
- 查看全员绩效报表

### 角色 B：店员
- 登录查看自己今日/本周排班和任务
- 查阅工作细则和考核标准（只读）
- 查看自己的考核结果：合格/不合格项、完成/未完成任务
- 查看个人历史绩效

## 三、核心功能模块（MVP）

### 模块 1：账号与权限
- 管理员账号：手机号 + 密码登录
- 店员账号：由管理员创建，手机号 + 密码登录
- 登录后自动跳转对应角色的首页


### 模块 2：工作细则库
**管理员操作：**
- 创建/编辑/删除工作细则条目
- 细则分类（如：开店流程、接待规范、商品陈列、收银操作、闭店流程等）
- 细则支持图文（文字 + 配图）

**店员操作：**
- 分类浏览所有工作细则
- 搜索关键词查找对应细则
- 只读，不可修改

### 模块 3：考核标准库
**管理员操作：**
- 创建考核项目（如：服务态度、陈列整洁度、开店是否准时、是否完成当日任务等）
- 每项考核设定：满分、评分说明、合格线
- 支持分类（日常行为类、月度目标类、特殊任务类）

**店员操作：**
- 查看所有考核项目标准（只读）
- 了解每项"合格"的定义

### 模块 4：排班与任务
**管理员操作：**
- 按日期、按门店创建排班（哪天谁上班）
- 为每个排班创建当日任务清单（如：整理B区陈列、补货、完成5条视频上传）
- 任务可以设定优先级（高/中/低）

**店员操作：**
- 登录后首页显示：今日是否排班、今日任务列表
- 任务可标记为「已完成」
- 查看本周排班日历

### 模块 5：考核打分
**管理员操作：**
- 选择店员 + 选择考核周期（日 / 周 / 月）
- 按考核标准库逐项打分
- 填写文字备注（具体指出问题或亮点）
- 提交后生成该店员的考核记录

**店员操作（只读）：**
- 查看管理员提交的考核结果
- 每项显示：得分 / 满分 / 合格/不合格标记
- 显示管理员备注
- 总结：哪些项合格，哪些项不合格，需要改进的地方

### 模块 6：绩效看板
**管理员看板：**
- 全员绩效排名
- 每人近期考核趋势图（折线）
- 任务完成率统计

**店员个人看板：**
- 本月考核总分 vs 上月对比
- 历次考核记录时间轴
- 任务完成率（本周 / 本月）

## 四、用户故事（核心场景）

| 场景 | 用户故事 |
|---|---|
| 新店员入职 | 作为新店员，我想登录系统查看开店流程细则，以便快速上手工作 |
| 每日上班 | 作为店员，我想一进系统就看到今天有哪些任务，以便合理安排工作顺序 |
| 完成任务 | 作为店员，我想勾选完成的任务，以便让管理员知道我做了什么 |
| 查看考核 | 作为店员，我想看到上次考核哪几项不合格，以便针对性改进 |
| 管理员打分 | 作为管理员，我想在周结束时对每个店员打分，以便有据可查地发绩效 |
| 管理员排班 | 作为管理员，我想一次性安排本周排班，以便店员提前知道自己几号上班 |
| 管理员看数据 | 作为管理员，我想看到所有店员的任务完成率，以便判断谁需要重点跟进 |

## 五、非功能需求

| 项目 | 要求 |
|---|---|
| 端 | 手机 H5 优先（店员用手机登录），管理员支持电脑端 |
| 响应速度 | 页面加载 < 2秒 |
| 数据安全 | 店员只能看到自己的考核数据，不能看其他人 |
| 离线能力 | 本期不做，纯在线 |
| 多门店 | 支持（清远店、其他城市店），数据互相隔离 |
